<idea>
  **Design Movement**: **Swiss Style (International Typographic Style)**
  
  **Core Principles**:
  1.  **Clareza e Legibilidade**: O foco principal é a apresentação clara dos dados financeiros.
  2.  **Grid Matemático**: Layouts estruturados rigidamente para transmitir precisão e confiança.
  3.  **Objetividade**: Remoção de elementos decorativos desnecessários; a forma segue a função.
  4.  **Contraste Forte**: Uso de tamanhos de fonte e pesos para criar hierarquia visual imediata.

  **Color Philosophy**:
  *   **Base**: Branco (#FFFFFF) e Cinza Neutro (#F5F5F5) para o fundo, garantindo que os dados sejam o protagonista.
  *   **Acento**: Vermelho Suíço (#FF3B30) para ações críticas e erros, e Azul Cobalto (#0047AB) para ações primárias e confiança.
  *   **Texto**: Preto Puro (#000000) ou Cinza Chumbo (#333333) para máximo contraste.
  *   *Intent*: Transmitir seriedade, precisão contábil e eficiência profissional.

  **Layout Paradigm**:
  *   Assimétrico mas equilibrado.
  *   Uso extensivo de espaço em branco para separar seções lógicas (Upload -> Processamento -> Resultado).
  *   Tabelas de dados com linhas finas e tipografia monoespaçada para números.

  **Signature Elements**:
  1.  **Tipografia Gigante**: Títulos grandes e em negrito (Helvetica ou similar) para guiar o usuário.
  2.  **Linhas Divisórias Espessas**: Uso de linhas pretas ou coloridas para separar cabeçalhos e seções.
  3.  **Ícones Geométricos**: Ícones simples e vetoriais, sem preenchimento ou detalhes excessivos.

  **Interaction Philosophy**:
  *   Feedback imediato e preciso. Botões mudam de estado de forma nítida (sem fades longos).
  *   Transições de "corte seco" ou deslizamentos rápidos e precisos.

  **Animation**:
  *   Movimentos lineares e rápidos (ease-out-quart).
  *   Elementos entram em cena deslizando de baixo para cima com opacidade.

  **Typography System**:
  *   **Display**: Inter (Bold/Black) ou Helvetica Neue (se disponível).
  *   **Body**: Inter (Regular) para texto e JetBrains Mono ou Roboto Mono para dados numéricos.
</idea>

<idea>
  **Design Movement**: **Neumorphism (Soft UI) Refined**
  
  **Core Principles**:
  1.  **Suavidade e Profundidade**: Elementos parecem extrudados do fundo, criando uma sensação tátil.
  2.  **Minimalismo Cromático**: Paleta quase monocromática para reduzir a carga cognitiva.
  3.  **Foco no Conteúdo**: O container é sutil, destacando os números e resultados.
  4.  **Acessibilidade Aprimorada**: Diferente do neumorfismo clássico, garantiremos contraste suficiente para leitura.

  **Color Philosophy**:
  *   **Base**: Off-White (#E0E5EC) ou Azul Pálido (#F0F4F8).
  *   **Sombras**: Combinação de sombras claras (brancas) e escuras (azul-acinzentado) para criar volume.
  *   **Acento**: Verde Menta (#00C896) para sucesso/crédito e Laranja Suave (#FF9F43) para atenção/débito.
  *   *Intent*: Tornar uma tarefa contábil tediosa em algo leve, moderno e agradável de usar.

  **Layout Paradigm**:
  *   Centralizado e flutuante. O "card" principal de processamento flutua no centro da tela.
  *   Bordas arredondadas generosas (20px+).

  **Signature Elements**:
  1.  **Botões "Soft"**: Botões que parecem pressionáveis fisicamente (efeito de baixo relevo ao clicar).
  2.  **Cards em Camadas**: Informações hierárquicas apresentadas em "placas" sobrepostas sutilmente.
  3.  **Gradientes Sutis**: Uso de gradientes lineares quase imperceptíveis para dar "brilho" às superfícies.

  **Interaction Philosophy**:
  *   Sensação tátil. Cliques têm feedback visual de "afundar".
  *   Drag-and-drop para upload com feedback visual elástico.

  **Animation**:
  *   Curvas de mola (spring physics) para movimentos naturais e orgânicos.
  *   Elementos "flutuam" suavemente ao aparecer.

  **Typography System**:
  *   **Display**: Nunito ou Quicksand (Rounded) para reforçar a suavidade.
  *   **Body**: DM Sans ou Lato para legibilidade limpa.
</idea>
</text>

<idea>
  **Design Movement**: **Cyberpunk / Data Terminal**
  
  **Core Principles**:
  1.  **Alta Densidade de Informação**: Estética inspirada em terminais financeiros e dashboards de trading.
  2.  **Modo Escuro Obrigatório**: Fundo escuro para reduzir cansaço visual e destacar dados coloridos.
  3.  **Tech-Oriented**: Visual que remete a processamento de dados de alta performance.
  4.  **Grid Modular**: Tudo se encaixa em blocos retangulares perfeitos.

  **Color Philosophy**:
  *   **Base**: Preto Profundo (#050505) ou Azul Noturno (#0A192F).
  *   **Acento**: Ciano Neon (#00FFFF) para elementos ativos e Magenta Neon (#FF00FF) para destaques.
  *   **Dados**: Verde Terminal (#00FF00) para valores positivos e Vermelho Laser (#FF0000) para negativos.
  *   *Intent*: Fazer o usuário se sentir um "hacker" ou analista de dados de elite.

  **Layout Paradigm**:
  *   Dashboard de tela cheia.
  *   Bordas finas e brilhantes (glow) separando os painéis.
  *   Fontes monoespaçadas em quase tudo.

  **Signature Elements**:
  1.  **Efeito Glitch/Scanline**: Sutis efeitos de varredura ou glitch em transições (opcional, sem prejudicar leitura).
  2.  **Bordas Brilhantes**: Containers com borders de 1px e leve box-shadow colorido.
  3.  **Micro-interações de Código**: Textos que aparecem como se estivessem sendo digitados.

  **Interaction Philosophy**:
  *   Instantâneo e responsivo.
  *   Sons de interface (opcional/sutil) ou feedback visual de "comando aceito".

  **Animation**:
  *   Aparecimento sequencial rápido (staggered).
  *   Efeitos de "piscar" cursor.

  **Typography System**:
  *   **Display**: Rajdhani ou Orbitron (Tech/Futuristic).
  *   **Body**: Fira Code, Space Mono ou IBM Plex Mono (Monospaced).
</idea>
